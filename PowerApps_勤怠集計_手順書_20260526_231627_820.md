# PowerApps 勤怠データ集計ツール 作成手順書（完全初心者向け・日本語環境）

## はじめに

この手順書は、プログラミング経験がない方でも **PowerApps** と **Power Automate** を使って勤怠データ集計ツールを自作できるように、画面キャプチャなしでも再現できるレベルで詳細に解説しています。

**完成するアプリの動き：**
SharePoint 上の北京・大連・河南の各拠点フォルダにある勤怠 Excel（1ヶ月分）を選択 → 拠点別パラメータ（昼休憩時間・基本勤務時間・残業閾値）を自動取得 → 出勤/退勤時刻から工数（0.5h単位）を自動計算 → 単価表とマッチングして単価を付与 → 氏名別シートに転記した月次集計ファイル（1ファイルに全データ集約）を自動作成

> **補足**：本ツールの入出力は**整月（1ヶ月分）データ**を前提としています。各拠点の入力 Excel は1ヶ月分の勤怠レコードを含み、出力ファイルも1ヶ月分の全データを1ファイルに集約します。

## Step 1：SharePoint 側の準備

### 1-1 SharePoint サイトにアクセスする

1. Web ブラウザで [office.com](https://www.office.com) を開き、職場アカウントで**サインイン**
2. 左上のアプリアンカー（点が9つ並んだアイコン）をクリック → 「**SharePoint**」を選択

> **画面の見方**：SharePoint のトップページが表示されます。左側に「ホーム」「自分のサイト」などのメニューがあります。

### 1-2 ドキュメント ライブラリを作成する

1. 左メニューから作成済みの**チーム サイト**をクリック（なければ「＋サイトの作成」で「チーム サイト」を新規作成）
2. サイトのトップページで「**＋ 新規**」→「**ドキュメント ライブラリ**」をクリック
3. 以下の設定でライブラリを作成：

| 設定項目 | 入力値 |
|---------|--------|
| 名前 | `勤怠データ` |
| 説明 | `各拠点の勤怠Excelファイル格納庫` |
| サイト ナビゲーションに表示 | はい（チェックを入れる） |

4. 「**作成**」ボタンをクリック

> **ここまでで**：サイト内に「勤怠データ」というドキュメント ライブラリが作成されました。

### 1-3 拠点フォルダを作成する

1. 「勤怠データ」ライブラリを開く
2. 「**＋ 新規**」→「**フォルダー**」をクリック
3. フォルダー名に `北京` と入力 → 「**作成**」
4. 同様に `大連`、`河南`、`集計結果` のフォルダも作成

**完成イメージ：**
```
勤怠データ/
├── 北京/
├── 大連/
├── 河南/
└── 集計結果/
```

### 1-4 勤怠 Excel テンプレートを準備する

各拠点で使う Excel のレイアウトは異なっていても構いませんが、**中身の情報**（どの列に何が入っているか）を事前に定義しておきます。

#### 入力用 Excel のサンプル（拠点：北京）

| 列 | A | B | C | D | E |
|----|---|---|---|---|---|
| 1行目 | **氏名** | **日付** | **出勤時刻** | **退勤時刻** | **作業番号** |
| 2行目 | 張三 | 2026/05/01 | 9:16 | 18:29 | PRJ-001 |
| 3行目 | 李四 | 2026/05/01 | 8:50 | 19:10 | PRJ-002 |
| 4行目 | 張三 | 2026/05/02 | 9:05 | 17:45 | PRJ-001 |

#### 入力用 Excel のサンプル（拠点：大連／列の並びが異なる）

| 列 | A | B | C | D | E |
|----|---|---|---|---|---|
| 1行目 | **日付** | **氏名** | **退勤時刻** | **出勤時刻** | **作業番号** |
| 2行目 | 2026/05/01 | 王五 | 18:00 | 8:30 | DL-101 |

> **重要なポイント**：拠点ごとに列の位置が違っても、後述の Power Automate で「この拠点なら氏名はA列、日付はB列…」とマッピングすれば対応できます。

> **補足（整月データ）**：各拠点の入力 Excel は**1ファイルに1ヶ月分**の勤怠レコードを格納してください。たとえば「北京_202605.xlsx」には北京拠点の5月分全データが入ります。これにより、フローはファイル単位で月初〜月末の全レコードを一括処理します。

#### Excel をテーブル化する（必須）

1. セル範囲（A1:E4 など）を選択
2. 「**ホーム**」タブ →「**テーブルとして書式設定**」をクリック
3. 好みのスタイルを選び、「**先頭行をテーブルの見出しとして使用する**」にチェック
4. 「**テーブル デザイン**」タブが表示されたら、左端の「**テーブル名**」を以下のように設定：

| 拠点 | テーブル名 |
|------|-----------|
| 北京 | `勤怠Table_北京` |
| 大連 | `勤怠Table_大連` |
| 河南 | `勤怠Table_河南` |

> **なぜテーブル化が必要？**：Power Automate は「テーブル」の形になっているデータしか読み取れません。ただのセル範囲ではエラーになります。

#### 出力用テンプレート Excel

`集計結果` フォルダに `勤怠集計_テンプレート.xlsx` を配置します。このファイルには**氏名ごとにシート**を用意します。

**シート名「張三」のレイアウト：**

| A | B | C | D | E | F | G | H | I | J |
|---|---|---|---|---|---|---|---|---|---|
| **日付** | **出勤時刻** | **退勤時刻** | **勤務時間** | **残業時間** | **作業番号** | **案件** | **自由入力１** | **自由入力２** | **単価** |

全シート同じレイアウトで、**必ずテーブル化**（テーブル名は `勤怠集計Table`）してください。各氏名のシートをあらかじめ作成しておきます。

> **注意**：出力テンプレートの見出し行（日付・出勤時刻・退勤時刻…の行）は**1行目にあるとは限らない**ケースを想定しています。後述の「項目名マッチング」方式により、見出し行が何行目にあっても正しく列位置を特定してデータを書き込みます。

> **補足（整月データ）**：出力ファイルは1ヶ月分の全データを1ファイルに集約します。各氏名のシートに、その月の全勤怠レコードが日付順に追記されます。

### 1-6 拠点別勤怠パラメータ設定（新規）

各拠点で勤務条件が異なるケースに対応するため、拠点別のパラメータファイルを用意します。

#### パラメータファイルの作成

1. 「勤怠データ」ライブラリの直下（ルート）に `拠点別パラメータ.xlsx` を配置します
2. 以下の列構成でテーブルを作成し、**テーブル名**を `勤怠パラメータTable` に設定：

| 列 | A | B | C | D |
|----|---|---|---|---|
| 1行目 | **拠点名** | **昼休憩時間(分)** | **基本勤務時間(h)** | **残業閾値(h)** |
| 2行目 | 北京 | 60 | 8 | 8.5 |
| 3行目 | 大連 | 60 | 8 | 8.5 |
| 4行目 | 河南 | 60 | 8 | 8.5 |

| 列名 | 説明 | 例 |
|------|------|-----|
| 拠点名 | 拠点識別子。Flow の `LocationName` と一致させる | `北京` |
| 昼休憩時間(分) | 勤務時間から控除する昼休憩の分数 | `60`（1時間） |
| 基本勤務時間(h) | 所定労働時間 | `8`（8時間） |
| 残業閾値(h) | この時間を超えた分が残業と判定される | `8.5`（8.5時間） |

> **補足**：拠点ごとに昼休憩時間や残業基準が異なる場合（例：大連は昼休憩90分、河南は残業閾値8.0hなど）、この表の値を変更するだけで対応できます。新しい拠点を追加する場合は、この表に行を追加してください。

#### Power Automate での読み取り（Flow 2 に追加）

Flow 2「BatchProcessAttendance」の冒頭（変数初期化の前）に、以下のステップを追加します：

**ステップ0-A：拠点別パラメータの読み取り**

1. 「**＋ 新しいステップ**」→ 検索で `excel online テーブル内の行` → 「**テーブル内の行を一覧表示**」
2. 設定：

| フィールド | 入力値 |
|-----------|--------|
| 場所 | SharePoint サイトURL |
| ドキュメント ライブラリ | `勤怠データ` |
| ファイル | `/勤怠データ/拠点別パラメータ.xlsx` |
| テーブル | `勤怠パラメータTable` |

**ステップ0-B：該当拠点のパラメータを抽出**

3. 「**＋ 新しいステップ**」→ 検索で `配列のフィルター処理` → 「**配列のフィルター処理**」
4. 設定：

| フィールド | 入力値 |
|-----------|--------|
| From | 動的コンテンツ「**value**」（テーブル内の行を一覧表示 から） |
| 条件（左辺） | `item()?['拠点名']` |
| 条件（演算子） | `が次の値に等しい` |
| 条件（右辺） | 動的コンテンツ「**LocationName**」（トリガー入力） |

**ステップ0-C：パラメータ値を変数化**

5. フィルタ結果の最初の行から各パラメータを取り出し、変数に格納：

```
// 式で first() を使って先頭行を取得
first(body('配列のフィルター処理'))?['昼休憩時間(分)']
first(body('配列のフィルター処理'))?['基本勤務時間(h)']
first(body('配列のフィルター処理'))?['残業閾値(h)']
```

6. 「**変数を初期化する**」で以下の変数を追加（既存の変数初期化ブロックに追加）：

| 変数名 | 種類 | 値 |
|--------|------|-----|
| `lunchBreakMinutes` | 整数 | `first(body('配列のフィルター処理'))?['昼休憩時間(分)']` |
| `baseWorkHours` | 浮動小数点数 | `first(body('配列のフィルター処理'))?['基本勤務時間(h)']` |
| `overtimeThreshold` | 浮動小数点数 | `first(body('配列のフィルター処理'))?['残業閾値(h)']` |

> **注意**：このパラメータ値は後続の Office Scripts 実行時に引数として渡します。

### 1-7 単価表の準備（新規）

各従業員の時間単価を管理するため、単価表ファイルを用意します。この単価表は退職者対応（退社日ありの行より退社日なしの行を優先）を自動的に処理します。

#### 単価表ファイルの作成

1. ローカルPCの `C:\勤怠データ\` フォルダ（なければ作成）に `単価表.xlsx` を配置します
   > **補足**：単価表は SharePoint 上の「勤怠データ」ライブラリ直下に配置しても構いません。その場合は Flow 内のファイルパスを SharePoint のパスに読み替えてください。

2. 以下の列構成でテーブルを作成し、**テーブル名**を `単価表Table` に設定：

| 列 | A | B | C |
|----|---|---|---|
| 1行目 | **氏名** | **単価** | **退社日** |
| 2行目 | 張三 | 2500 | |
| 3行目 | 李四 | 3000 | |
| 4行目 | 王五（退職済） | 2000 | 2025/12/31 |
| 5行目 | 王五（現行） | 2800 | |

| 列名 | 説明 | 備考 |
|------|------|------|
| 氏名 | 従業員氏名。出力テンプレートのシート名と一致させる | 文字列 |
| 単価 | 時間単価（円） | 数値 |
| 退社日 | 退職日（退職済みの場合のみ記入） | 日付。空欄＝現役 |

> **重要なルール**：同一氏名が退社日あり／なしの両方で登録されている場合、**退社日が空欄の行を優先**してマッチングします。これは「現行の単価を優先する」という意味です。過去の退職者データは退社日入りで残しておき、新しい契約の行を空欄で追加してください。

#### Power Automate での読み取り（Flow 2 に追加）

Flow 2「BatchProcessAttendance」の変数初期化後、件名別パラメータ読み取りに続けて以下のステップを追加します：

**ステップ0-D：単価表の読み取り**

1. 「**＋ 新しいステップ**」→ 検索で `sharepoint ファイル コンテンツ` → 「**ファイル コンテンツの取得**」
2. 設定：

| フィールド | 入力値 |
|-----------|--------|
| サイト アドレス | SharePoint サイトURL |
| ファイル パス | `/勤怠データ/単価表.xlsx` |

3. 「**＋ 新しいステップ**」→ `excel online` → 「**テーブル内の行を一覧表示**」
4. 設定：

| フィールド | 入力値 |
|-----------|--------|
| 場所 | SharePoint サイトURL |
| ドキュメント ライブラリ | `勤怠データ` |
| ファイル | 動的コンテンツ「**Id**」（ファイル コンテンツの取得から） |
| テーブル | `単価表Table` |

**ステップ0-E：退社日空欄を優先した単価マップの作成**

5. 「**＋ 新しいステップ**」→ 検索で `選択` → 「**選択**」アクション
6. **開始**：動的コンテンツ「**value**」（テーブル内の行を一覧表示 から）
7. **マップ**（スイッチモード：テキストモード）：

| キー | 値（式タブ） |
|------|-------------|
| `氏名` | `item()?['氏名']` |
| `単価` | `item()?['単価']` |
| `退社日` | `item()?['退社日']` |

8. 「**＋ 新しいステップ**」→ `変数` → 「**変数を初期化する**」
9. 名前：`priceMapJson`、種類：`文字列`
10. 値：以下の式（式タブ）で退社日空欄を優先した JSON を生成：

```
// 注意：Power Automate 標準関数では複雑な重複排除ロジックが難しいため、
// 後続の Office Scripts 側で退社日空欄優先ロジックを実装します。
// ここでは単価表の全行を JSON 文字列として変数に格納し、
// Office Scripts に引数として渡します。
string(body('選択'))
```

> **補足**：単価表の JSON は後続の Office Scripts に引数として渡し、スクリプト内で「退社日が空欄の行を優先する」マッチングロジックを実装します（後述）。

## Step 2：Power Automate フローの作成

Power Automate（パワーオートメイト）は「裏方」です。PowerApps から呼び出されて SharePoint のファイルを読み書きします。

### 2-A：フロー1「GetExcelFilesByLocation」を作る

**目的**：PowerApps で選択された拠点のフォルダ内にある Excel ファイル一覧を返す

#### 手順

1. [make.powerautomate.com](https://make.powerautomate.com) にアクセス
2. 左メニュー「**作成**」→「**インスタント クラウド フロー**」をクリック
3. フロー名に `GetExcelFilesByLocation` と入力
4. トリガーの一覧から「**PowerApps (V2)**」を選択 → 「**作成**」

> **画面の見方**：白いキャンバスに「PowerApps (V2)」というトリガーが1つ表示されました。ここにステップを追加していきます。

#### ステップ1：入力パラメータを追加

1. 「PowerApps (V2)」トリガーをクリック
2. 「**＋入力の追加**」→「**テキスト**」を選択
3. 入力名に `LocationName` と入力

#### ステップ2：「フォルダー内のファイルを一覧表示」を追加

1. 「**＋ 新しいステップ**」をクリック
2. 検索ボックスに `sharepoint フォルダー内のファイルを一覧` と入力
3. 「**フォルダー内のファイルを一覧表示**」アクションを選択
4. 以下のように設定：

| フィールド | 入力値 |
|-----------|--------|
| サイト アドレス | あなたの SharePoint サイトのURL（例：`https://xxxx.sharepoint.com/sites/YourSite`） |
| フォルダー パス | `/勤怠データ/` の右側にある雷アイコンをクリック → 動的コンテンツから `LocationName` を選択 |

> **雷アイコンとは？**：テキストボックスの右端にある稲妻マークです。これをクリックすると、前のステップから受け取った値を埋め込めます。

#### ステップ3：必要な情報だけを抽出（「選択」アクション）

1. 「**＋ 新しいステップ**」→ 検索で `選択` → 「**選択**」アクション
2. **開始**：動的コンテンツから「**value**」（フォルダー内のファイル一覧 から）を選択
3. **マップ**（「キー」と「値」のペアを入力）：

| キー | 値（雷アイコンから選択） |
|------|------------------------|
| `Name` | `表示名` |
| `Id` | `Id` |

> **「選択」アクションの意味**：ファイル一覧にはたくさんの情報が含まれていますが、アプリに必要なのは「ファイル名」と「ファイルID」だけ。ここで必要な列だけに絞っています。

#### ステップ4：PowerApps に結果を返す

1. 「**＋ 新しいステップ**」→ 検索で `応答` → 「**PowerApps または Flow に応答する**」
2. 「**＋ 出力の追加**」→「**テキスト**」
3. タイトル：`files`、値：動的コンテンツから「**body**」（選択アクション）を選択

#### 完成イメージ

```
PowerApps(V2) トリガー
  ├── 入力: LocationName（テキスト）
  │
  ├── フォルダー内のファイルを一覧表示
  │    サイト: https://xxx.sharepoint.com/sites/xxx
  │    フォルダー: /勤怠データ/{LocationName}
  │
  ├── 選択
  │    開始: value（↑の結果）
  │    マップ: Name=表示名, Id=Id
  │
  └── PowerApps または Flow に応答する
       出力: files = body（選択の結果）
```

4. 右上の「**保存**」をクリック
5. 左上の「←」矢印でフロー一覧に戻る

### 2-B：フロー2「BatchProcessAttendance」を作る

#### ステップ3：工数計算ロジック（ここが一番重要）

ループの外に戻って（Apply to each の後）、計算用の処理を追加します。Power Automate 標準の数式だけでは複雑な時刻計算が難しいため、**Office Scripts** を使う方法をお勧めします。

##### Office Scripts を使う（推奨）

1. Excel for Web で適当なブックを開く
2. 「**自動化**」タブ →「**新しいスクリプト**」
3. 以下のコードを貼り付けて「**スクリプト名**」を `CalculateWorkHours` に変更 →「**保存**」

```typescript
function main(
  workbook: ExcelScript.Workbook,
  attendanceJson: string,
  lunchBreakMinutes: number = 60,
  baseWorkHours: number = 8,
  overtimeThreshold: number = 8.5,
  priceTableJson: string = "[]"
): string {
  // 勤怠データのパース
  const records: Array<{
    氏名: string;
    日付: string;
    出勤時刻: string;
    退勤時刻: string;
    作業番号: string;
  }> = JSON.parse(attendanceJson);

  // 単価表のパース → 退社日が空欄の行を優先
  const priceRaw: Array<{
    氏名: string;
    単価: number;
    退社日: string;
  }> = JSON.parse(priceTableJson);

  // 退社日空欄を優先したマッピング（同名が複数ある場合、退社日空欄の方を採用）
  const priceMapTemp: { [name: string]: { 単価: number; 退社日: string } } = {};
  for (const p of priceRaw) {
    const existing = priceMapTemp[p.氏名];
    if (!existing) {
      // 初出：そのまま登録
      priceMapTemp[p.氏名] = { 単価: p.単価, 退社日: p.退社日 || "" };
    } else if (!p.退社日 || p.退社日 === "") {
      // 既存より退社日空欄の行を優先（上書き）
      priceMapTemp[p.氏名] = { 単価: p.単価, 退社日: "" };
    }
    // 既存が空欄で新着が退社日あり → 上書きしない（空欄優先）
  }

  // 最終的な氏名→単価マップ（退社日情報は除外）
  const priceMap: { [name: string]: number } = {};
  for (const [name, info] of Object.entries(priceMapTemp)) {
    priceMap[name] = info.単価;
  }

  const results: Array<{
    氏名: string;
    日付: string;
    出勤時刻: string;
    退勤時刻: string;
    出勤時刻_修正: string;
    退勤時刻_修正: string;
    勤務時間: number;
    残業時間: number;
    作業番号: string;
    単価: number;
  }> = [];

  for (const record of records) {
    // 時刻文字列 "9:16" を時間と分に分解
    const [inH, inM] = record.出勤時刻.split(':').map(Number);
    const [outH, outM] = record.退勤時刻.split(':').map(Number);

    // 0.5時間単位で切り捨て
    const inRounded = inH + Math.floor(inM / 30) * 0.5;
    const outRounded = outH + Math.floor(outM / 30) * 0.5;

    // 修正後時刻を文字列に戻す
    const inH2 = Math.floor(inRounded);
    const inM2 = (inRounded - inH2) * 60;
    const outH2 = Math.floor(outRounded);
    const outM2 = (outRounded - outH2) * 60;

    const 出勤時刻_修正 = `${String(inH2).padStart(2, '0')}:${String(inM2).padStart(2, '0')}`;
    const 退勤時刻_修正 = `${String(outH2).padStart(2, '0')}:${String(outM2).padStart(2, '0')}`;

    // 勤務時間 = (退勤 - 出勤) - 昼休憩時間（パラメータから取得）
    const lunchBreakHours = lunchBreakMinutes / 60;
    const workMinutes = (outRounded - inRounded) * 60 - lunchBreakMinutes;
    const 勤務時間 = Math.max(0, workMinutes / 60);

    // 残業計算（残業閾値パラメータを使用）
    let 残業時間 = 0;
    if (勤務時間 > overtimeThreshold) {
      const overtime = 勤務時間 - overtimeThreshold;
      残業時間 = Math.ceil(overtime * 2) / 2; // 0.5単位で切り上げ
    }
    // 基本勤務時間超〜残業閾値以下は残業0（グレーゾーン）

    // 単価マッチング（氏名でルックアップ、見つからない場合は0）
    const 単価 = priceMap[record.氏名] || 0;

    results.push({
      氏名: record.氏名,
      日付: record.日付,
      出勤時刻: record.出勤時刻,
      退勤時刻: record.退勤時刻,
      出勤時刻_修正,
      退勤時刻_修正,
      勤務時間: Math.round(勤務時間 * 10) / 10, // 小数点1桁
      残業時間: Math.round(残業時間 * 10) / 10,
      作業番号: record.作業番号,
      単価: 単価
    });
  }

  return JSON.stringify(results);
}
```

> **補足（パラメータの変更点）**：
> - `lunchBreakMinutes`（昼休憩時間(分)）：拠点別パラメータファイルから取得。デフォルト60分。
> - `baseWorkHours`（基本勤務時間(h)）：所定労働時間。デフォルト8h。
> - `overtimeThreshold`（残業閾値(h)）：この値を超えた分が残業。デフォルト8.5h。
> - `priceTableJson`（単価表JSON）：単価表の全行データ。スクリプト内で退社日空欄優先マッチングを実施。

##### 項目名マッチングのロジック（2つ目の Office Scripts）

見出し行の位置が不定でも正しく書き込めるよう、以下の方式で列マッピングを実行します。

**方式の概要：**
1. テンプレートコピー後、各シートを走査
2. 「日付」「出勤時刻」「退勤時刻」などの**項目名を含む行**を見出し行として特定
3. 見出し行の各列セル値から列インデックスをマッピング（例：「日付」→A列、「出勤時刻」→C列…）
4. 見出し行の**次の行**からデータを書き込み開始

4. Excel for Web で新しくスクリプトを作成し、**スクリプト名**を `WriteToTemplateWithMapping` に設定：

```typescript
function main(
  workbook: ExcelScript.Workbook,
  resultJson: string
): void {
  // 計算結果のパース
  const results: Array<{
    氏名: string;
    日付: string;
    出勤時刻_修正: string;
    退勤時刻_修正: string;
    勤務時間: number;
    残業時間: number;
    作業番号: string;
    案件: string;
    自由入力１: string;
    自由入力２: string;
    単価: number;
  }> = JSON.parse(resultJson);

  // 項目名 → 書き込む値のフィールド名 のマッピング定義
  const columnDefinitions: Array<{ header: string; field: string }> = [
    { header: "日付", field: "日付" },
    { header: "出勤時刻", field: "出勤時刻_修正" },
    { header: "退勤時刻", field: "退勤時刻_修正" },
    { header: "勤務時間", field: "勤務時間" },
    { header: "残業時間", field: "残業時間" },
    { header: "作業番号", field: "作業番号" },
    { header: "案件", field: "案件" },
    { header: "自由入力１", field: "自由入力１" },
    { header: "自由入力２", field: "自由入力２" },
    { header: "単価", field: "単価" }
  ];

  // 氏名でグループ化
  const grouped: { [name: string]: typeof results } = {};
  for (const r of results) {
    if (!grouped[r.氏名]) {
      grouped[r.氏名] = [];
    }
    grouped[r.氏名].push(r);
  }

  // 各氏名シートを処理
  for (const [name, records] of Object.entries(grouped)) {
    const sheet = workbook.getWorksheet(name);
    if (!sheet) {
      console.log(`シート "${name}" が見つかりません。スキップします。`);
      continue;
    }

    // シートをアクティブ化
    sheet.activate();

    // 見出し行の検出：全行を走査して項目名を含む行を探す
    const usedRange = sheet.getUsedRange();
    if (!usedRange) {
      console.log(`シート "${name}" にデータがありません。`);
      continue;
    }

    const rowCount = usedRange.getRowCount();
    const colCount = usedRange.getColumnCount();
    let headerRowIndex = -1;
    const columnMap: { [field: string]: number } = {};

    // 行を走査して見出し行を特定
    for (let row = 0; row < rowCount; row++) {
      const rowValues = usedRange.getRow(row).getValues()[0];
      let matchCount = 0;
      const tempMap: { [field: string]: number } = {};

      for (let col = 0; col < colCount; col++) {
        const cellValue = String(rowValues[col] || "").trim();
        for (const def of columnDefinitions) {
          if (cellValue === def.header) {
            tempMap[def.field] = col;
            matchCount++;
            break;
          }
        }
      }

      // 過半数の項目名がマッチしたら見出し行と判定
      if (matchCount >= columnDefinitions.length * 0.5) {
        headerRowIndex = row;
        // 見つかったマッピングを columnMap にコピー
        for (const [field, colIdx] of Object.entries(tempMap)) {
          columnMap[field] = colIdx;
        }
        break;
      }
    }

    if (headerRowIndex === -1) {
      console.log(`シート "${name}" で見出し行が見つかりませんでした。`);
      continue;
    }

    console.log(`シート "${name}"：見出し行 ${headerRowIndex + 1} 行目を検出しました。`);

    // 見出し行の次の行からデータを書き込み（headerRowIndex + 1）
    const dataStartRow = headerRowIndex + 1;

    // 日付順にソート
    records.sort((a, b) => a.日付.localeCompare(b.日付));

    // データ行を構築
    const dataRows: (string | number)[][] = [];
    for (const r of records) {
      const row: (string | number)[] = new Array(colCount).fill("");
      for (const def of columnDefinitions) {
        const colIdx = columnMap[def.field];
        if (colIdx !== undefined) {
          row[colIdx] = (r as any)[def.field] !== undefined ? (r as any)[def.field] : "";
        }
      }
      dataRows.push(row);
    }

    // シートにデータを書き込み
    if (dataRows.length > 0) {
      const targetRange = sheet.getRangeByIndexes(
        dataStartRow, 0, dataRows.length, colCount
      );
      targetRange.setValues(dataRows);
    }
  }
}
```

> **項目名マッチングのポイント**：
> - シートの全行を走査し、`columnDefinitions` で定義した項目名（「日付」「出勤時刻」など）を含む行を探します。
> - 過半数（50%以上）の項目名が一致した行を「見出し行」と判定。
> - 見出し行の列位置をマッピングし、その**次の行**からデータを書き込みます。
> - これにより、テンプレートに見出し行以外のヘッダー情報（タイトル行やサマリ行）が含まれていても正しく動作します。

```
  ├── 配列のフィルター処理（拠点名で絞込）
  ├── 変数 lunchBreakMinutes / baseWorkHours / overtimeThreshold
  │
  ├── 【単価表読取】ファイル コンテンツの取得（単価表.xlsx）
  ├── テーブル内の行を一覧表示（単価表Table）
  ├── 選択（氏名/単価/退社日）
  ├── 変数 priceTableJson
  │
  ├── 変数 allAttendanceData / processedNames / fileIdArray
  │
  ├── Apply to each（ファイルID配列）
  │     ├── ファイル コンテンツの取得
  │     ├── テーブル内の行を一覧表示
  │     └── Apply to each（各行）
  │           └── 配列変数に追加（allAttendanceData）
  │
  ├── Office Scripts 実行：CalculateWorkHours
  │    （attendanceJson / lunchBreakMinutes / baseWorkHours / overtimeThreshold / priceTableJson）
  │
  ├── ファイルのコピー（テンプレート → 新ファイル）
  │
  ├── 【項目名マッチング書込】Office Scripts 実行：WriteToTemplateWithMapping
  │    （resultJson = CalculateWorkHours の結果 + 案件名等）
  │
  └── PowerApps または Flow に応答する
       出力: resultFileUrl
```

```powerapps
                    LocationName: loc,
                    ProjectName: prj,

                    FreeInput2: f2
                })
```

| テスト花子 | 05/02 | 8.0h | 0h | 3000 | 9:30〜18:30 → 8h勤務、グレーゾーンに入らず |

#### 項目名マッチングテスト

出力テンプレートの見出し行が**2行目以降**にあるケースをテストしてください。例：

| 行 | A | B | C | D |
|----|---|---|---|---|
| 1 | **勤怠集計表（2026年5月）** | | | |
| 2 | **日付** | **出勤時刻** | **退勤時刻** | **勤務時間** | ... |
| 3 | （データ書き込み開始行） | | | |

この場合、1行目はタイトル行として無視され、2行目が見出し行として検出されます。データは3行目から書き込まれます。

<a id="step5"></a>
## Step 5：運用とメンテナンス

### 5-1 アプリの公開（他の人に使ってもらう）

1. PowerApps 編集画面の右上「**保存**」→「**公開**」
2. 「**公開**」ボタンをクリック
3. 公開後、「**共有**」から同僚のメールアドレスを入力して共有
4. 共有された人は [make.powerapps.com](https://make.powerapps.com) または Teams 内からアプリを利用可能

### 5-2 拠点の追加

新しい拠点（例：上海）を追加する場合：

1. SharePoint に `勤怠データ/上海/` フォルダを作成
2. Excel テンプレートを配置・テーブル化
3. `拠点別パラメータ.xlsx` に上海の行を追加（昼休憩時間・基本勤務時間・残業閾値を設定）
4. PowerApps の `DdLocation` の Items に `"上海"` を追加
5. Power Automate フロー2に上海用の条件分岐を追加（テーブル名の切り替え）
6. 出力用テンプレートに上海の従業員シートを追加
7. `単価表.xlsx` に上海の従業員の単価行を追加

### 5-3 拠点別パラメータの変更

各拠点の勤務条件が変わった場合：

1. SharePoint 上の `拠点別パラメータ.xlsx` を開く
2. 該当拠点の行の値を更新（例：昼休憩時間を60→90に変更）
3. ファイルを保存して SharePoint にアップロードし直す
4. 次回のフロー実行から自動的に新しいパラメータが適用されます

> **補足**：パラメータファイルはフロー実行のたびに読み取られるため、Power Automate フローの再公開は不要です。ファイルを更新するだけで即時反映されます。

### 5-4 単価表のメンテナンス

1. 従業員の単価が変わった場合：該当行の「単価」列を更新
2. 従業員が退職した場合：該当行の「退社日」列に退職日を入力（単価行は残す）
3. 再雇用などで単価が変わった場合：**新しい行**を追加し、退社日は空欄にする（空欄行が優先されるため）
4. 新入社員が入った場合：新しい行を追加（氏名・単価を入力、退社日は空欄）

### 5-5 定期的なバックアップ

- SharePoint 上の「勤怠データ」ライブラリは自動バージョン管理されています
- 必要に応じて「集計結果」フォルダ内の古いファイルを手動整理してください

## 補足：トラブルシューティング FAQ

### Q1：Power Automate のフローが動かない／エラーになる

**確認手順：**
1. [make.powerautomate.com](https://make.powerautomate.com) → 左メニュー「**マイ フロー**」
2. 該当フローの実行履歴（28日間）を確認
3. 失敗している場合は実行をクリック → エラーになったアクションを特定

**よくある原因：**

| エラー | 原因 | 解決策 |
|--------|------|--------|
| `NotFound` | ファイルパスが間違っている | フォルダーパスが `/勤怠データ/北京` など正しいか確認 |
| `Unauthorized` | SharePoint への接続が切れている | フロー編集画面で各 SharePoint アクションの「…」→「＋新しい接続の追加」で再接続 |
| `BadRequest`（テーブル） | Excel がテーブル化されていない | Excel を開き「ホーム」→「テーブルとして書式設定」を適用 |
| `タイムアウト` | ファイル数が多すぎる | ファイルを10件以下に分割して実行 |

### Q2：PowerApps でフローが呼び出せない

1. フローとアプリが**同じ環境**にあるか確認（PowerApps 編集画面の右上に環境名が表示されています）
2. フロー名を正しく入力しているか → 数式バーに `フロー名.` と打つと候補が出るはず
3. フローが「オン」になっているか → Power Automate で確認

### Q3：ギャラリーにファイル一覧が表示されない

1. SharePoint フォルダに Excel ファイルが実際にあるか確認
2. `DdLocation` の OnChange が正しく設定されているか
3. ブラウザの開発者ツール（F12）→「コンソール」でエラーを確認

### Q4：工数計算結果がおかしい

1. 入力 Excel の時刻形式が `hh:mm`（例：`09:16`）になっているか確認
   - セルの書式設定で「ユーザー定義」→ `h:mm` に
2. Office Scripts の計算ロジックを再確認
3. `拠点別パラメータ.xlsx` の値（昼休憩時間・残業閾値）が正しいか確認
4. 手計算で検算し、期待値と比較

### Q5：単価が出力されない／間違っている

1. `単価表.xlsx` のテーブル名が `単価表Table` になっているか確認
2. 氏名が単価表と出力テンプレートのシート名で完全一致しているか（全角/半角、スペースの有無）
3. 退社日が空欄の行が優先されるため、同名で退社日入りの行が先にマッチしていないか確認
4. Power Automate の実行履歴で単価表読み取りステップが成功しているか確認

### Q6：出力テンプレートの見出し行が正しく検出されない

1. テンプレートの見出し項目名が Office Scripts の `columnDefinitions` と**完全一致**しているか確認（全角/半角）
2. 最低でも過半数（5項目以上）の項目名が存在するか確認
3. シートがテーブル化されているか確認（テーブル化されていると見出し行検出が安定します）

### Q7：日本語が文字化けする

1. Excel ファイルが UTF-8 で保存されているか確認
2. Power Automate の「ファイル コンテンツの取得」後、適切なエンコードで読み取っているか

### Q8：拠点別パラメータが反映されない

1. `拠点別パラメータ.xlsx` の拠点名が Flow の `LocationName`（"北京""大連""河南"）と完全一致しているか
2. Power Automate の「配列のフィルター処理」で正しい列名（`拠点名`）を指定しているか
3. フィルタ結果が空になっていないか（実行履歴で確認）
